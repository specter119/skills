# Framework Selection

## Goals and Frameworks

| Goal | Framework | Reference |
| --- | --- | --- |
| Persuade management | SCQA / Pyramid Principle | Barbara Minto, McKinsey |
| Academic paper | IMRaD | General academic writing conventions |
| Technical documentation | Inverted pyramid | Journalism / Divio |
| Business proposal | Problem-Solution-Impact | Consulting and business writing |
| Case study | STAR | Behavioral case expression |

## Writing Quality Guardrails

- **MECE**: Arguments are mutually exclusive and collectively exhaustive
- **So What**: Every section answers “what does this mean?”
- **Data-backed**: Key judgments are supported by data, citations, or examples
- **Reader-aware**: Adjust terminology and depth to match the reader’s background

## When to Change Framework

- You realize mid-draft that sections feel like material dumps rather than arguments
- The user adds new goals or the target audience shifts
- The data cannot support the current structure
