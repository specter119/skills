# Design System

在写任何一页之前，先定视觉系统。这样能降低认知负担，也能提升后续迭代效率。

## 1. 排印层级

建议最多使用 2 个字体族：

- Sans：适合屏幕演示
- Mono：仅用于代码或术语

推荐优先级：

- Sans: `Inter`, `Noto Sans`, `Source Sans 3`, `Arial`
- Serif: `Source Serif 4`, `Noto Serif`, `Georgia`
- Mono: `JetBrains Mono`, `Fira Code`, `Consolas`

### 推荐字号

| 层级 | 用途 | 推荐尺寸 |
| --- | --- | --- |
| Display | 封面 / 转场 | 32-48pt |
| H1 | 页面标题 | 20-24pt |
| H2 | 小节标题 | 16-18pt |
| Body | 正文 | 14-16pt |
| Caption | 注释 / 来源 | 10-12pt |
| Code | 代码块 | 9-11pt |

### 层级行为

字号只是层级的一个维度。每页的排印层级需要满足三个条件：

1. **一个主导入口**。视觉上最先看到的只有一个元素——不是两个、不是三个。
2. **层级间有意的节奏**。相邻层级之间要有足够的对比，不能只差 2pt。
3. **信息流可恢复**。即使出现层级倒置（如正文比标题大），读者仍能通过阅读顺序理解结构。

建立层级的五个向量——至少让两个同向作用于主导元素：

| 向量 | 控制什么 | 方向 |
| --- | --- | --- |
| 字号 | 大小对比 | 大 → 小 = 主 → 次 |
| 字重 | 质量对比 | 重 → 轻 = 主 → 次 |
| 间距 | 呼吸空间 | 大间距 = 更重要 |
| 字距 | 紧凑 vs 松散 | 紧 = 快；宽 = 仪式感 |
| 对齐 | 与网格的关系 | 打破对齐 = 标记重要性 |

### 三级工作模型

大多数 slide 页面可以映射到三个功能层级：

| 层级 | 角色 | 典型实现 |
| --- | --- | --- |
| Primary | 入口点，每页只有一个 | 字号 + 间距，或对齐突破 |
| Secondary | 结构支撑 | 字重变化或字号台阶 |
| Tertiary | 附属信息（标签、注释） | 字号缩小、字重降低 |

折叠面以上出现超过三个视觉层级，通常是版式问题而不是层级机会。

### 排印反模式

- 梯级字重（regular → medium → semibold → bold），看起来像默认梯度而非设计意图
- 所有 section 间距一样，间距不传递层级信息
- 只靠字号做层级，字重、间距、对齐全部统一

### 基础规则

- 标题行高 1.2-1.3
- 正文行高 1.4-1.6
- 强调用字重优先，不依赖颜色
- 全大写字母必须加 ≥0.06em 字距

## 2. 色彩纪律

### 色板结构

一个可控的色板有四个层次，先规划再写样式：

| 层次 | 像素占比 | 角色 |
| --- | --- | --- |
| 中性色 | 70-90% | 背景、表面、文字、边框 |
| 强调色（仅一个） | 5-10% | 仅 accent——不要发明第二个强调色 |
| 语义色 | 0-5% | success / warn / error |
| 效果 | <1% | 渐变、发光，极少使用 |

### 语义命名

按用途命名，不按色相：

- ✅ `c-primary`, `c-accent`, `c-success`
- ❌ `c-blue-500`, `c-green-600`

至少定义：`primary`、`accent`、`success`、`error`、`muted`、`surface`、`background`、`text`

### 强调色纪律

AI 生成界面最常见的可读性问题是强调色过度使用。硬性上限：

- **每屏最多 2 处可见的 accent 使用**。典型组合：一个眉标/标签 + 一个主 CTA；或一个高亮卡片 + 一个选中标签。
- 如果同屏有 CTA 和超链接，超链接降级为下划线而非 accent 颜色。

### 对比度门限

作为通过/不通过的门限检查：

| 组合 | 最低对比度 |
| --- | --- |
| 正文（≤16pt）在背景上 | 4.5:1 |
| 大字（>18pt 或 14pt bold）在背景上 | 3:1 |
| UI 元素在相邻表面上 | 3:1 |

经验上，`#757575` 在白底通常比 `#999999` 安全。

### 深色主题

避免纯黑和纯白——都会引起视觉振动和疲劳：

| 角色 | 深色主题 | 浅色主题 |
| --- | --- | --- |
| 背景 | `#0f0f0f`（非 `#000`） | `#FAFAFA`（非 `#FFF`） |
| 前景 | `#f0f0f0`（非 `#FFF`） | `#212121`（非 `#000`） |

深色表面上优先用**半透明白色边框**而非实色深边框——1px `white.transparentize(92%)` 能暗示结构而不增加视觉噪音。

### 避免 AI 默认色

- **不要无意使用 indigo 系**（`#6366f1`、`#4f46e5`、`#8b5cf6` 等）。这是最可靠的 AI 生成标记。如果设计需要 indigo，必须是用户明确要求。
- **不要在首页使用双色"信任"渐变**（紫→蓝、蓝→青、紫→粉）。平面 + 排印层级几乎总是更好的选择。

## 3. 密度与风格

根据 deck 类型选择密度：

| 类型 | 特征 | 建议 |
| --- | --- | --- |
| 演讲型 | 一页一个观点，依赖讲者口播 | 留白大、信息轻 |
| 商业型 | 需要独立阅读 | 结构清晰、信息更密 |
| 技术型 | 代码、架构、流程较多 | 混合密度，强调图示和证据 |

### 风格选择

可借鉴 `pptx-generator` 的风格分层：

- Sharp: 适合数据密集、正式报告
- Soft: 适合大多数商业 deck
- Rounded: 适合产品介绍、较友好的视觉语气

### 视觉节奏

连续页面需要密度和骨架变化。经验规则：

- 连续 3 页不要使用同一种视觉骨架
- 有意交替——一页紧凑，一页留白——让节奏感受到设计意图
- 对称的等分布局（dashboard 式 2×2 grid）在 slide 中几乎总是失败

## 4. 反 AI 默认值

AI 生成 slide 时容易落入的模式——检测到这些信号时应该主动修正：

### P0（必须修正）

1. **捏造指标**——"10× faster"、"99.9% uptime"、"用户节省 4 小时/周"。没有数据源就不要编数字，用 `—` 或标注 `[待补数据]`。
2. **填充文字**——lorem ipsum、"Feature One / Two / Three"、"seamless innovation"、"streamline your workflow"。空缺的内容是需要用版式解决的设计问题，不是靠编词填满。
3. **表情符号当图标**——`✨ 🚀 🎯 ⚡ 🔥 💡` 出现在标题或列表中。用文字层级或简洁的形状代替。
4. **投影不可读的对比度**——不满足 §2 对比度门限。

### P1（应当修正）

5. 标准化流程 "Cover → Agenda → Features → Summary" 完全没有变化。至少引入一个非常规板块（全幅引用、内嵌 mini-demo、状态对比时间线）。
6. 默认调色板——未经修改的 Tailwind/Material 色板直接使用。
7. 每屏 accent 超过 2 处。
8. 装饰性渐变或 blob/wave 背景——没有功能目的的几何装饰。

### P2（可以改善）

9. 完美对称布局，没有视觉张力。
10. 所有 card 组件形状相同（圆角 + 左边框 accent）。

## 7. 常见失误

- 只定义颜色，不定义层级和间距
- 所有页都用同一套卡片布局
- 代码块不做深浅对比，导致投影难读
- 用 Web UI 思维做 slide，出现 dashboard 式平均分栏
- 梯级字重代替有意的层级跳跃
- 间距统一，不传递结构信息
- 强调色泛滥，超过 2 处/屏
